= 从零搭建 Antora + AsciiDoc + antora-ui-spring 文档站点

本指南详细介绍如何从零开始搭建一个完整的 Antora 文档站点，使用 AsciiDoc 编写内容，搭配 antora-ui-spring 主题，并集成本地搜索、代码高亮、图表渲染等功能。

== 前置条件

=== Node.js 18+

Antora 3.x 要求 Node.js 18 或更高版本。推荐使用 https://github.com/nvm-sh/nvm[nvm] 管理 Node.js 版本：

[source,bash]
----
# 安装 nvm（如果尚未安装）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 重新加载 shell 配置
source ~/.bashrc   # 或 source ~/.zshrc

# 安装并使用 Node.js 18 LTS
nvm install 18
nvm use 18

# 验证版本
node --version   # 应输出 v18.x.x 或更高
npm --version    # 应输出 9.x.x 或更高
----

TIP: 也可以使用 https://github.com/Schniz/fnm[fnm] 作为 nvm 的替代方案，速度更快。

=== Git

确保已安装 Git 并完成基本配置：

[source,bash]
----
git --version    # 应输出 git version 2.x.x

# 基本配置（如尚未配置）
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
----

=== npm

npm 随 Node.js 一起安装。确保版本 >= 9：

[source,bash]
----
npm --version
----

== 初始化项目

=== 创建目录结构

[source,bash]
----
# 创建项目根目录
mkdir antora-asciidoc-demo
cd antora-asciidoc-demo

# 创建文档目录结构
mkdir -p docs/modules/ROOT/pages
mkdir -p docs/modules/ROOT/images
mkdir -p docs/modules/ROOT/examples
mkdir -p docs/modules/ROOT/partials
mkdir -p supplemental-ui/css

# 创建 .gitignore
cat > .gitignore << 'EOF'
node_modules/
build/
.cache/
EOF
----

----

│   ├── antora.yml             # 组件描述符

npm init -y

=== 配置 package.json scripts

编辑 `package.json`，添加构建脚本：

[source,json]
----
{
  "name": "antora-asciidoc-demo",
  "version": "1.0.0",
  "description": "Antora + AsciiDoc + antora-ui-spring 示例站点",
  "private": true,
  "scripts": {
    "build": "antora antora-playbook.yml",
    "build:local": "antora antora-playbook.yml --to-dir=build/site",
    "preview": "npx http-server build/site -p 8080 -o",
    "clean": "rm -rf build .cache"
  },
  "dependencies": {
    "@antora/cli": "^3.1",
    "@antora/site-generator": "^3.1",
    "@antora/lunr-extension": "^1.0",
    "asciidoctor-kroki": "^0.18"
  }
}
----

脚本说明：

* `npm run build` -- 默认构建，输出到 Antora 默认目录
* `npm run build:local` -- 构建到 `build/site` 目录
* `npm run preview` -- 启动本地 HTTP 服务器预览站点
* `npm run clean` -- 清理构建产物和缓存

== 创建 Antora Playbook (antora-playbook.yml)

Playbook 是 Antora 的核心配置文件，定义了站点的所有行为。在项目根目录创建 `antora-playbook.yml`：

[source,yaml]
----
# antora-playbook.yml - Antora 站点主配置
site:
  title: Antora AsciiDoc 示例站点       # <1>
  url: https://example.com/docs          # <2>
  start_page: docs::index.adoc           # <3>

content:
  sources:
    - url: .                              # <4>
      branches: HEAD                      # <5>
      start_path: docs                    # <6>

ui:
  bundle:
    url: https://github.com/spring-io/antora-ui-spring/releases/download/latest/ui-bundle.zip  # <7>
    snapshot: true                        # <8>
  supplemental_files: ./supplemental-ui   # <9>

asciidoc:
  attributes:
    source-highlighter: highlight.js      # <10>
    source-linenums-option: ''            # <11>
  extensions:
    - asciidoctor-kroki                   # <12>

antora:
  extensions:
    - require: '@antora/lunr-extension'   # <13>
      languages:
        - en
        - zh                              # <14>
----
<1> 站点标题，显示在页面顶部导航栏
<2> 站点部署后的基础 URL，用于生成绝对链接和 sitemap
<3> 站点默认起始页，格式为 `组件名::页面ID`
<4> 内容源 URL，`.` 表示当前本地仓库
<5> 要使用的分支，`HEAD` 表示当前工作目录的内容（适合本地开发）
<6> 文档组件在仓库中的起始路径
<7> UI bundle 下载地址，这里使用 Spring 官方的 Antora UI 主题
<8> `snapshot: true` 表示每次构建都重新下载 UI bundle（适合使用 `latest` 标签）
<9> 补充 UI 文件目录，用于覆盖默认样式
<10> 代码块使用 highlight.js 进行语法高亮
<11> 全局启用代码块行号（值为空字符串表示布尔型属性设为 true）
<12> 注册 asciidoctor-kroki 扩展以支持图表渲染
<13> 注册 Lunr 搜索扩展
<14> 搜索支持的语言列表，`zh` 为中文分词支持

=== site 配置详解

[source,yaml]
----
site:
  title: 我的文档站点
  url: https://docs.example.com
  start_page: docs::index.adoc
  # 可选：站点 robots.txt 配置
  # robots: allow
  # 可选：生成 sitemap
  # keys:
  #   google_analytics: UA-XXXXXXXX-X
----

IMPORTANT: `url` 字段在生产部署时必须设置为实际部署地址，否则搜索索引和站点地图中的链接会不正确。

=== content sources 配置

内容源支持本地目录和远程 Git 仓库：

[source,yaml]
----
content:
  sources:
    # 本地仓库（开发时使用）
    - url: .
      branches: HEAD
      start_path: docs

    # 远程 Git 仓库
    # - url: https://github.com/your-org/your-repo.git
    #   branches: [main, v1.x, v2.x]
    #   start_path: docs

    # 多组件场景：同一仓库不同路径
    # - url: .
    #   branches: HEAD
    #   start_path: docs-en
----

=== UI bundle 配置

antora-ui-spring 是 Spring 官方维护的 Antora UI 主题，特点包括：

* 清爽的设计风格
* 响应式布局
* 内置搜索框位置
* 版本切换器
* 面包屑导航

[source,yaml]
----
ui:
  bundle:
    url: https://github.com/spring-io/antora-ui-spring/releases/download/latest/ui-bundle.zip
    snapshot: true
  supplemental_files: ./supplemental-ui
  # 可选：输出目录
  # output_dir: _
----

NOTE: 如果网络环境无法直接访问 GitHub Releases，可以将 UI bundle 下载到本地，使用本地路径：`url: ./ui-bundle.zip`

== 创建文档组件 (docs/antora.yml)

组件描述符 `antora.yml` 定义了一个文档组件的元数据：

[source,yaml]
----
name: docs                  # <1>
title: 文档示例              # <2>
version: '2.0'              # <3>
nav:                        # <4>
  - modules/ROOT/nav.adoc
----
<1> 组件名称，用于跨组件引用（如 `docs::page.adoc`）
<2> 组件显示标题，出现在 UI 的组件选择器中
<3> 版本号，必须用引号包裹数字（避免 YAML 解析为浮点数）
<4> 导航文件列表，按顺序显示在侧边栏

=== 组件描述符完整选项

[source,yaml]
----
name: docs
title: 文档示例
version: '2.0'
# 版本显示名称（可选，默认使用 version 值）
display_version: '2.0 稳定版'
# 预发布标记（可选）
# prerelease: Beta
# 组件起始页（可选，默认为 ROOT:index.adoc）
start_page: index.adoc
# AsciiDoc 属性（组件级别）
asciidoc:
  attributes:
    # 这里定义的属性仅在本组件中生效
    project-name: 示例项目
    project-version: '2.0.0'
nav:
  - modules/ROOT/nav.adoc
----

=== 关于 version 字段

* 版本号支持语义化版本：`'1.0'`、`'2.1.3'`
* 特殊值 `~` (tilde) 表示无版本（versionless），URL 中不包含版本段
* 特殊值 `true` 使用分支名作为版本号

== 编写文档页面

=== 目录结构约定

[source]
----
docs/modules/ROOT/
├── nav.adoc           # 侧边栏导航定义
├── pages/             # 文档页面（.adoc 文件）
│   ├── index.adoc     # 组件首页
│   ├── setup-guide.adoc
│   └── api-reference.adoc
├── images/            # 图片资源
│   └── architecture.png
├── examples/          # 示例代码文件（可通过 include 引入）
│   └── HelloWorld.java
├── partials/          # 可复用的文档片段
│   └── common-note.adoc
└── attachments/       # 可下载的附件
    └── sample.zip
----

=== nav.adoc 导航文件格式

导航文件使用 AsciiDoc 无序列表语法，支持多级嵌套：

[source,asciidoc]
----
* xref:index.adoc[首页]
* xref:setup-guide.adoc[构建指南]
* 高级主题
** xref:i18n.adoc[国际化]
** xref:multi-version.adoc[多版本管理]
** xref:deployment.adoc[部署]
* 参考
** xref:api-reference.adoc[API 参考]
** https://docs.antora.org[Antora 官方文档^]
----

说明：

* `+xref:page.adoc[显示文本]+` -- 引用同模块内的页面
* `+xref:module:page.adoc[显示文本]+` -- 引用同组件其他模块的页面
* `+xref:component:module:page.adoc[显示文本]+` -- 跨组件引用
* 纯文本（不带 xref）作为分组标题
* 缩进层级使用 `**` 表示子项

=== 页面基本结构

每个 `.adoc` 文件的基本结构：

[source,asciidoc]
----
= 页面标题
:description: 页面描述，用于 SEO
:keywords: antora, asciidoc, 文档
:page-layout: default

正文内容从这里开始。

== 二级标题

段落文本...

=== 三级标题

更多内容...
----

常用页面属性：

[cols="1,2"]
|===
|属性 |说明

|`:description:`
|页面描述，用于 HTML meta 标签

|`:keywords:`
|关键词，用于 SEO

|`:page-layout:`
|页面布局模板（如 `home`、`default`）

|`:toc:`
|是否显示页内目录（`left`、`right`、`auto`）

|`:sectnums:`
|是否显示章节编号
|===

== 配置本地搜索

=== 方案一：Pagefind（推荐）

https://pagefind.app[Pagefind] 是一个高性能的静态站点搜索工具，相比 Lunr 有显著优势：

* **按需加载**：初始仅加载 ~5KB 入口文件，索引分片按需获取，而非一次性加载全部索引
* **内置 CJK 支持**：自动处理中文/日文/韩文分词，无需额外依赖
* **构建无侵入**：作为后处理步骤运行，不需要 Antora 扩展
* **Component UI**：内置模态框搜索界面，支持键盘快捷键（`/` 打开，`Escape` 关闭）

==== 安装

[source,bash]
----
npm install -D pagefind
----

==== 构建流程

Pagefind 在 Antora 构建完成后运行，对生成的 HTML 建立索引：

[source,bash]
----
# 第一步：Antora 构建站点
npx antora antora-playbook.yml --to-dir=build/site

# 第二步：Pagefind 索引（仅索引文章正文）
npx pagefind --site build/site --root-selector "article.doc"
----

`--root-selector "article.doc"` 确保只索引文档正文内容，排除导航栏、页头、页脚等。

==== package.json 配置

[source,json]
----
{
  "scripts": {
    "build": "antora antora-playbook.yml --to-dir=build/site && npx pagefind --site build/site --root-selector article.doc",
    "serve": "npx http-server build/site -p 8888"
  },
  "devDependencies": {
    "pagefind": "^1.3.0"
  }
}
----

==== UI 集成（Component UI 模态框）

在 Handlebars 模板中引入 Pagefind Component UI：

.supplemental-ui/partials/footer-scripts.hbs
[source,html]
----
<link rel="stylesheet" href="{{{siteRootPath}}}/pagefind/pagefind-component-ui.css">
<script src="{{{siteRootPath}}}/pagefind/pagefind-component-ui.js"></script>
<pagefind-config pagefind-path="{{{siteRootPath}}}/pagefind/"></pagefind-config>
<pagefind-modal>
  <pagefind-modal-header>
    <pagefind-input></pagefind-input>
  </pagefind-modal-header>
  <pagefind-modal-body>
    <pagefind-results></pagefind-results>
  </pagefind-modal-body>
  <pagefind-modal-footer>
    <pagefind-keyboard-hints></pagefind-keyboard-hints>
  </pagefind-modal-footer>
</pagefind-modal>
----

在导航栏中添加搜索触发按钮：

.supplemental-ui/partials/nav-search.hbs
[source,html]
----
<pagefind-modal-trigger>搜索</pagefind-modal-trigger>
----

Component UI 内置以下键盘快捷键：

|快捷键 |功能

运行 `npx pagefind` 后会在站点目录下生成 `pagefind/` 目录：

----

=== 方案二：@antora/lunr-extension

Lunr 是 Antora 官方提供的搜索扩展，构建时生成完整索引文件并在浏览器端加载。

NOTE: Lunr 会一次性加载全部索引（大型站点可达数百 KB），且中文支持需要额外安装 `@node-rs/jieba` 原生依赖。推荐优先使用 <<方案一：Pagefind（推荐）,Pagefind>>。

==== 安装

[source,bash]
----
npm install @antora/lunr-extension@^1.0
# 中文分词支持（可选）
npm install @node-rs/jieba
----

==== Playbook 中配置

在 `antora-playbook.yml` 的 `antora.extensions` 部分添加：

[source,yaml]
----
antora:
  extensions:
    - require: '@antora/lunr-extension'
      languages:
        - en
        - zh
----

==== 中文搜索支持

Lunr.js 默认仅支持英文分词。添加 `zh` 语言后，扩展会使用 `lunr-languages` 包中的中文分词器（基于 nodejieba）。

[IMPORTANT]
====
中文搜索分词质量依赖于 `lunr-languages` 中的中文支持实现。对于大型中文文档站点，如果搜索效果不理想，可考虑：

* 在文档中为关键术语添加英文别名
* 使用 Pagefind 替代（内置更好的 CJK 分词）
====

搜索功能的工作原理：

1. 构建时，Lunr 扩展扫描所有页面内容
2. 生成搜索索引文件（`search-index.js`）
3. 浏览器端加载索引文件，用户输入关键词即可搜索
4. 无需服务端支持，纯前端实现

==== 搜索配置选项

[source,yaml]
----
antora:
  extensions:
    - require: '@antora/lunr-extension'
      languages:
        - en
        - zh
      # 可选：自定义索引字段权重
      # index_latest_only: true   # 仅索引最新版本
      # snippet_length: 200       # 搜索结果摘要长度
----

== 配置国际化 (i18n)

=== 多组件方案

Antora 的国际化推荐使用多组件方案：每种语言作为一个独立的文档组件。

[source]
----
project-root/
├── docs/                # 中文文档（默认语言）
│   ├── antora.yml       # name: docs
│   └── modules/ROOT/pages/
├── docs-en/             # 英文文档
│   ├── antora.yml       # name: docs-en
│   └── modules/ROOT/pages/
└── antora-playbook.yml
----

=== 创建英文组件

创建 `docs-en/antora.yml`：

[source,yaml]
----
name: docs-en
title: Documentation (English)
version: '2.0'
nav:
  - modules/ROOT/nav.adoc
----

创建对应的目录结构：

[source,bash]
----
mkdir -p docs-en/modules/ROOT/pages
----

创建 `docs-en/modules/ROOT/nav.adoc`：

[source,asciidoc]
----
* xref:index.adoc[Home]
* xref:getting-started.adoc[Getting Started]
----

=== Playbook 中添加多个 content sources

[source,yaml]
----
content:
  sources:
    # 中文文档
    - url: .
      branches: HEAD
      start_path: docs
    # 英文文档
    - url: .
      branches: HEAD
      start_path: docs-en
----

=== 在页面间添加语言切换链接

在中文页面顶部：

[source,asciidoc]
----
TIP: English version: xref:docs-en::index.adoc[English]
----

在英文页面顶部：

[source,asciidoc]
----
TIP: 中文版本: xref:docs::index.adoc[中文]
----

== 配置多版本

=== 多 start_path 方案

对于同一仓库中维护多个版本的场景：

[source]
----
project-root/
├── docs/                # 当前版本 (2.0)
│   └── antora.yml       # version: '2.0'
├── docs-v1/             # 旧版本 (1.0)
│   └── antora.yml       # version: '1.0'
└── antora-playbook.yml
----

Playbook 配置：

[source,yaml]
----
content:
  sources:
    - url: .
      branches: HEAD
      start_path: docs
    - url: .
      branches: HEAD
      start_path: docs-v1
----

[source,yaml]

* `main` 分支: `version: '3.0'`
* `v2.x` 分支: `version: '2.0'`
* `v1.x` 分支: `version: '1.0'`

=== 版本切换器

antora-ui-spring 主题内置了版本切换器组件。当同一组件存在多个版本时，页面顶部会自动显示版本下拉菜单，用户可以在不同版本间切换。

版本排序规则：

* 语义化版本号自动降序排列

=== CI/CD 集成

==== GitHub Actions 工作流示例

创建 `.github/workflows/docs.yml`：

[source,yaml]
----
name: Build and Deploy Docs

on:
  push:
    branches: [main]
    paths:
      - 'docs/**'
      - 'antora-playbook.yml'
      - 'package.json'
      - 'supplemental-ui/**'
  pull_request:
    branches: [main]
    paths:
      - 'docs/**'

# 设置 GITHUB_TOKEN 权限
permissions:
  contents: read
  pages: write
  id-token: write

# 只允许一个并发部署
concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0    # 需要完整历史记录以支持多分支版本

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Build site
        run: npm run build:local

      - name: Upload artifact
        if: github.ref == 'refs/heads/main'
        uses: actions/upload-pages-artifact@v3
        with:
          path: build/site

  deploy:
    if: github.ref == 'refs/heads/main'
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
----

==== 工作流说明

[cols="1,2"]
|===
|配置项 |说明

|`on.push.paths`
|仅在文档相关文件变更时触发构建

|`fetch-depth: 0`
|获取完整 Git 历史，支持多分支版本构建

|`npm ci`
|使用 `package-lock.json` 确保依赖版本一致

|`upload-pages-artifact`
|上传构建产物到 GitHub Pages

|`deploy-pages`
|将产物部署到 GitHub Pages 环境
|===

NOTE: 使用此工作流前，需要在仓库 Settings -> Pages 中将 Source 设置为 "GitHub Actions"。

== 常见问题

=== 图片路径问题

**问题：** 图片无法显示，控制台报 404 错误。

**原因：** Antora 有严格的资源目录约定。

**解决方案：**

1. 图片必须放在 `modules/ROOT/images/` 目录下
2. 在文档中使用 `image::filename.png[]` 引用（不需要路径前缀）
3. 跨模块引用使用 `image::module:filename.png[]`

[source,asciidoc]
----
// 正确 - 引用同模块 images 目录下的图片
image::architecture.png[架构图,width=600]

// 正确 - 指定图片标题
.系统架构图
image::architecture.png[架构图]

// 错误 - 不要使用相对路径
image::../images/architecture.png[]  // 这样不行！
----

=== 搜索不生效

**问题：** 搜索框无反应或搜索结果为空。

==== 使用 Pagefind 时

1. 确认构建后执行了 Pagefind 索引步骤：
+
[source,bash]
----
npx pagefind --site build/site --root-selector "article.doc"
----

2. 确认构建产物中存在 `pagefind/` 目录：
+
[source,bash]
----
ls build/site/pagefind/pagefind-entry.json
----

3. 确认 HTML 中 `pagefind-path` 属性指向正确路径。

4. 搜索仅在 HTTP 服务器中工作，直接用浏览器打开 `file://` 协议的 HTML 文件时搜索无法正常运行。

==== 使用 Lunr 时

1. 确认 `@antora/lunr-extension` 已正确安装：
+
[source,bash]
----
npm ls @antora/lunr-extension
----

2. 确认 Playbook 中的扩展配置正确（注意 YAML 缩进）：
+
[source,yaml]
----
antora:
  extensions:
    - require: '@antora/lunr-extension'
      languages: [en, zh]
----

3. 确认构建产物中存在搜索索引：
+
[source,bash]
----
ls build/site/search-index.js
----

4. 清理缓存重新构建：
+
[source,bash]
----
npm run clean && npm run build:local
----

5. 搜索仅在 HTTP 服务器中工作，直接用浏览器打开 `file://` 协议的 HTML 文件时搜索无法正常运行。

=== 样式覆盖方法

**问题：** 想修改主题的默认样式，但直接改 UI bundle 不现实。

**解决方案：** 使用 `supplemental-ui` 机制：

1. 在 `supplemental-ui/css/site.css` 中编写覆盖样式
2. 使用浏览器开发者工具找到目标元素的选择器
3. 通过更高优先级的选择器覆盖

[source,css]
----
/* 示例：修改顶部导航栏背景色 */
.navbar {
  background-color: #1a1a2e !important;
}

/* 示例：修改链接颜色 */
.doc a {
  color: #0066cc;
}

/* 示例：隐藏版本选择器 */
.version-menu {
  display: none;
}
----

TIP: 尽量避免使用 `!important`，优先通过增加选择器特异性来覆盖。

=== 版本不显示

**问题：** 页面上不显示版本切换器。

**原因：** 版本切换器仅在同一组件存在多个版本时才会显示。

**解决方案：**

1. 确认至少有两个版本的组件被正确加载
2. 确认不同版本的 `antora.yml` 中 `name` 相同但 `version` 不同
3. 检查 content sources 是否正确包含了多个版本

[source,yaml]
----
# 确保两个版本的 name 相同
# docs/antora.yml
name: docs          # <-- 相同
version: '2.0'

# docs-v1/antora.yml

=== 构建报错：找不到 start_page

**问题：** 构建时报错 `Start page specified for site not found`。

**解决方案：**

1. 确认 `start_page` 格式正确：`组件名::页面ID`
2. 确认目标页面文件确实存在
3. 组件名必须与 `antora.yml` 中的 `name` 一致

[source,yaml]
----
# antora-playbook.yml
site:
  start_page: docs::index.adoc  # 组件名为 "docs"，页面为 "index.adoc"

# docs/antora.yml 中必须有
name: docs   # 与 start_page 中的组件名对应
----

=== Kroki 图表渲染失败

**问题：** 图表显示为空白或报错。

**排查步骤：**

1. 确认网络可访问 https://kroki.io
2. 检查图表语法是否正确（先在 https://kroki.io 在线编辑器中验证）
3. 如果使用自建 Kroki 服务器，确认服务正常运行
4. 检查构建日志中是否有错误信息：
+
[source,bash]
----
npx antora --log-level=debug antora-playbook.yml --to-dir=build/site 2>&1 | grep -i kroki
----

=== 中文文件名问题

**问题：** 使用中文命名的 `.adoc` 文件导致链接异常。

**解决方案：** 始终使用英文 + 连字符命名文件，在文件内部设置中文标题：

[source,asciidoc]
----
// 文件名: getting-started.adoc
= 快速开始

内容...
----

== 文本格式

=== 基本格式

以下展示 AsciiDoc 支持的各种行内文本格式：

* *粗体文本* — 使用单个星号包裹
* **粗体短语**内嵌在词中 — 使用双星号
* _斜体文本_ — 使用单个下划线包裹
* __斜体短语__内嵌在词中 — 使用双下划线
* `等宽文本` — 使用反引号包裹
* ``等宽短语``内嵌在词中 — 使用双反引号
* #高亮文本# — 使用井号包裹
* ##高亮短语##内嵌在词中 — 使用双井号
* ^上标文本^ — 例如：E=mc^2^
* ~下标文本~ — 例如：H~2~O
* [.line-through]#删除线文本# — 使用角色标记
* [.underline]#下划线文本# — 使用角色标记

=== 组合格式

可以将多种格式组合使用：

* *_粗斜体_* — 粗体加斜体
* `*等宽粗体*` — 等宽加粗体
* `_等宽斜体_` — 等宽加斜体
* *#粗体高亮#* — 粗体加高亮

== 代码块

=== Java 代码（带行号和 callout）

[source,java,linenums]
----
package com.example.service;

import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Service // <1>
public class OrderService {

    private final OrderRepository orderRepository;
    private final PaymentClient paymentClient;

    public OrderService(OrderRepository orderRepository,
                        PaymentClient paymentClient) { // <2>
        this.orderRepository = orderRepository;
        this.paymentClient = paymentClient;
    }

    @Transactional // <3>
    public Order createOrder(CreateOrderRequest request) {
        Order order = Order.builder()
                .userId(request.getUserId())
                .items(request.getItems())
                .totalAmount(calculateTotal(request.getItems()))
                .status(OrderStatus.PENDING)
                .build();

        order = orderRepository.save(order); // <4>

        paymentClient.initiatePayment(order.getId(), order.getTotalAmount()); // <5>

        return order;
    }
}
----
<1> 使用 `@Service` 注解标记为 Spring Bean
<2> 通过构造函数注入依赖，推荐方式
<3> `@Transactional` 确保数据一致性
<4> 持久化订单到数据库
<5> 异步调用支付服务发起支付

#!/bin/bash
# 自动部署脚本

set -euo pipefail

APP_NAME="my-service"
DEPLOY_DIR="/opt/apps/${APP_NAME}"
BACKUP_DIR="/opt/backup/${APP_NAME}"

echo ">>> 开始部署 ${APP_NAME}..."

# 备份当前版本
if [ -d "${DEPLOY_DIR}" ]; then
    TIMESTAMP=$(date +%Y%m%d_%H%M%S)
    cp -r "${DEPLOY_DIR}" "${BACKUP_DIR}/${TIMESTAMP}"
    echo ">>> 已备份到 ${BACKUP_DIR}/${TIMESTAMP}"
fi

# 拉取最新代码并构建
cd /opt/src/${APP_NAME}
git pull origin main
./gradlew clean build -x test

# 停止旧服务
systemctl stop ${APP_NAME} || true

# 部署新版本
cp build/libs/${APP_NAME}-*.jar "${DEPLOY_DIR}/${APP_NAME}.jar"

# 启动服务
systemctl start ${APP_NAME}

echo ">>> 部署完成！"
----

=== Python

[source,python]
----
from dataclasses import dataclass
from typing import List, Optional
import asyncio
import httpx

@dataclass
class TaskResult:
    task_id: str
    status: str
    data: Optional[dict] = None

async def fetch_tasks(base_url: str, task_ids: List[str]) -> List[TaskResult]:
    """并发获取多个任务的结果"""
    async with httpx.AsyncClient(timeout=30.0) as client:
        tasks = [
            client.get(f"{base_url}/api/tasks/{task_id}")
            for task_id in task_ids
        ]
        responses = await asyncio.gather(*tasks, return_exceptions=True)

    results = []
    for task_id, response in zip(task_ids, responses):
        if isinstance(response, Exception):
            results.append(TaskResult(task_id=task_id, status="error"))
        else:
            data = response.json()
            results.append(TaskResult(
                task_id=task_id,
                status=data["status"],
                data=data.get("result")
            ))
    return results

if __name__ == "__main__":
    results = asyncio.run(fetch_tasks(
        "https://api.example.com",
        ["task-001", "task-002", "task-003"]
    ))
    for r in results:
        print(f"[{r.status}] {r.task_id}: {r.data}")
----

=== SQL

[source,sql]
----
-- 创建订单表
CREATE TABLE IF NOT EXISTS t_order (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    order_no    VARCHAR(64)  NOT NULL COMMENT '订单号',
    user_id     BIGINT       NOT NULL COMMENT '用户ID',
    total_amount DECIMAL(10,2) NOT NULL DEFAULT 0.00 COMMENT '总金额',
    status      TINYINT      NOT NULL DEFAULT 0 COMMENT '状态: 0-待支付, 1-已支付, 2-已发货, 3-已完成',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_order_no (order_no),
    INDEX idx_user_id (user_id),
    INDEX idx_status_created (status, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='订单表';

-- 查询用户最近30天订单统计
SELECT
    u.username,
    COUNT(o.id) AS order_count,
    SUM(o.total_amount) AS total_spent
FROM t_user u
LEFT JOIN t_order o ON u.id = o.user_id
    AND o.created_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY u.id, u.username
HAVING order_count > 0
ORDER BY total_spent DESC
LIMIT 20;
----

== Include 指令

`include` 指令用于将其他文件的内容包含到当前文档中。

=== 基本用法

[source,asciidoc]
----
\include::partial$common-header.adoc[]
----

=== 包含部分内容（按标签）

在被包含的文件中使用标签标记：

[source,java]
----
// tag::service-method[]
public void processOrder(Order order) {
    validate(order);
    save(order);
    notify(order);
}
// end::service-method[]
----

然后在文档中只包含该标签的内容：

[source,asciidoc]
----
\include::example$OrderService.java[tag=service-method]
----

=== 包含指定行范围

[source,asciidoc]
----
\include::example$config.yaml[lines=5..15]
----

NOTE: 在 Antora 中，`include` 路径使用资源 ID 格式，如 `partial$`、`example$` 等前缀来指定文件所在的目录。

== 条件处理

AsciiDoc 支持条件指令，根据属性是否定义来包含或排除内容。

[source,asciidoc]
----
\ifdef::env-production[]
本内容仅在 `env-production` 属性被设置时显示。
\endif::[]
----

=== ifndef / endif

[source,asciidoc]
----
\ifndef::skip-tests[]
本内容在 `skip-tests` 属性 *未* 被设置时显示。
\ifndef::[]
----

[source,asciidoc]

\ifeval::["{backend}" == "html5"]

\endif::[]
----

TIP: 条件处理在多版本文档、多环境配置场景中非常有用。例如可以根据 `env-dev` 或 `env-prod` 属性来决定是否显示调试信息。

* 作者：{author}
* 日期：{revdate}

=== 常用文字替换

AsciiDoc 会自动处理以下替换：

* 版权符号：(C)
* 注册商标：(R)
* 商标符号：(TM)
* 省略号：...
* 破折号：--
* 右箭头：->
* 左箭头：<-
* 双右箭头：=>

=== 阻止替换

使用反斜杠可以阻止替换：\(C) 不会被转换。

使用直通（passthrough）宏：pass:[<strong>直接输出 HTML</strong>]

=== 行内直通

* `+文本不做任何处理+` — 加号包裹的内容原样输出
* pass:[*这不是粗体*] — pass 宏内容不做 AsciiDoc 处理

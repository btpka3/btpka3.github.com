= 从零搭建指南
:toc: macro

本文档记录了从零搭建 Docusaurus + AsciiDoc 静态文档站点的完整流程，涵盖以下功能：

* AsciiDoc 文档支持
* PlantUML / Mermaid 图表渲染
* Light / Dark 主题切换
* 浏览器端全文搜索
* i18n 国际化（中文 / English）
* 多版本文档切换
* 宽屏显示优化

toc::[]

== 环境要求

* Node.js >= 20（Node 22/24 与 Docusaurus 3.x 存在兼容问题）
* npm >= 9

== 第一步：初始化项目

[source,bash]
----
mkdir docusaurus-asciidoc-demo && cd docusaurus-asciidoc-demo
npm init -y
npm install @docusaurus/core@3.7.0 @docusaurus/preset-classic@3.7.0 react react-dom
npm install @asciidoc-js/docusaurus-asciidoc
----

=== 关键：webpack 版本锁定

Docusaurus 3.7.0 依赖的 webpackbar 与 webpack 5.106+ 不兼容（ProgressPlugin 参数校验报错），需要在 `package.json` 中锁定：

[source,json]
----
{
  "overrides": {
    "webpack": "5.97.1"
  }
}
----

=== 关键：不使用 `"type": "module"`

`package.json` 中 **不要** 设置 `"type": "module"`，否则 Docusaurus SSR 构建会报 `require.resolveWeak is not a function` 错误。需要 ESM 语法的配置文件使用 `.mjs` 扩展名。

== 第三步：PlantUML 支持（Kroki）

Docusaurus 本身不支持 PlantUML。通过自定义 remark 插件将 PlantUML 代码块转为 kroki.io 的 SVG 图片 URL。

=== remark-plantuml.mjs

[source,javascript]
----
import { visit } from 'unist-util-visit';
import { deflateSync } from 'node:zlib';

function encodeKroki(source, language) {
  const compressed = deflateSync(Buffer.from(source, 'utf-8'));
  return `https://kroki.io/${language}/svg/${compressed.toString('base64url')}`;
}

export default function remarkPlantuml() {
  return (tree) => {
    visit(tree, 'code', (node, index, parent) => {
      if (!parent || index == null || node.lang !== 'plantuml') return;
      const url = encodeKroki(node.value, 'plantuml');
      parent.children[index] = {
        type: 'paragraph',
        children: [{ type: 'image', url, alt: 'PlantUML Diagram' }],
      };
    });
  };
}
----

安装依赖：

[source,bash]
----
npm install unist-util-visit
----

=== Mermaid 支持

Mermaid 由 `@docusaurus/theme-mermaid` 提供客户端渲染，无需额外配置。在 AsciiDoc 中使用：

[source,asciidoc]
----
 [source,mermaid]

     A[Start] --> B[End]

== 第四步：修复 AsciiDoc 插件的二次编译问题

`@asciidoc-js/docusaurus-asciidoc@0.2.0` 有一个 bug：remark 插件在第一次编译后会删除注册表条目，导致第二次编译（Docusaurus 会分别构建 Server 和 Client）时内容为空。

=== 使用 patch-package 修复

[source,bash]
----
npm install patch-package
----

在 `package.json` 中添加 postinstall：

[source,json]
----
{
  "scripts": {
    "postinstall": "patch-package"
  }
}
----

创建 `patches/@asciidoc-js+docusaurus-asciidoc+0.2.0.patch`：

[source,diff]
----
--- a/node_modules/@asciidoc-js/docusaurus-asciidoc/lib/remark-plugin.js
+++ b/node_modules/@asciidoc-js/docusaurus-asciidoc/lib/remark-plugin.js
@@ -xx,xx +xx,xx @@
-    tree.children = mdastTree.children;
-    mdastRegistry.delete(filePath);
+    tree.children = JSON.parse(JSON.stringify(mdastTree.children));
----

核心原因：用深拷贝代替直接引用赋值，且不再删除注册表，确保多次编译都能读取到内容。

=== 修复代码块行号与表格合并

`@asciidoc-js/adast-util-to-mdast@1.0.0` 的 mdast 转换存在两个缺陷：

1. **代码块行号丢失**：AsciiDoc 的 `[source,java,linenums]` 中的 `linenums` 属性未传递给 Docusaurus（需要 `meta: "showLineNumbers"`）
2. **表格 colspan/rowspan 丢失**：`3+|` 等合并单元格语法被展开为多个空单元格，未生成 HTML `colspan` 属性

创建 `patches/@asciidoc-js+adast-util-to-mdast+1.0.0.patch` 修复：

[source,diff]
----
--- a/node_modules/@asciidoc-js/adast-util-to-mdast/dist/index.js
+++ b/node_modules/@asciidoc-js/adast-util-to-mdast/dist/index.js

 // handleListing: 添加行号支持
     const result = {
         type: 'code',
         value: listing.value,
         lang: listing.language ?? null,
     };
+    const attrs = listing.attributes ?? {};
+    if (attrs.linenums != null || attrs['linenums-option'] != null) {
+        result.meta = 'showLineNumbers';
+    }
     state.patch(node, result);

 // handleTable: 检测合并单元格
-    if (tableHasBlockCells(table)) {
+    if (tableHasBlockCells(table) || tableHasMergedCells(table)) {
         return buildRichTable(state, table);
     }

+// 新增函数：检测 colspan/rowspan
+function tableHasMergedCells(table) {
+    for (const section of table.children) {
+        for (const row of section.children) {
+            for (const cell of row.children) {
+                if ((cell.colspan && cell.colspan > 1) ||
+                    (cell.rowspan && cell.rowspan > 1))
+                    return true;
+            }
+        }
+    }
+    return false;
+}

 // buildRichTable: 输出 colspan/rowspan 属性
             for (const cell of row.children) {
                 const cellChildren = buildRichCellChildren(state, cell);
+                const hProperties = {};
+                if (cell.colspan && cell.colspan > 1)
+                    hProperties.colSpan = cell.colspan;
+                if (cell.rowspan && cell.rowspan > 1)
+                    hProperties.rowSpan = cell.rowspan;
                 cells.push({
                     type: 'richTableCell',
-                    data: { hName: cellTag },
+                    data: { hName: cellTag, hProperties },
                     children: cellChildren,
                 });
----

== 第五步：Light / Dark 主题

`docusaurus.config.mjs` 中 `themeConfig.colorMode` 配置即可启用主题切换按钮（导航栏自动出现）：

[source,javascript]
----
colorMode: {
  defaultMode: 'light',
  disableSwitch: false,          // false = 显示切换按钮
  respectPrefersColorScheme: true, // 跟随系统偏好
}
----

在 `src/css/custom.css` 中通过 `[data-theme='dark']` 选择器为暗色模式提供样式覆盖。

== 第七步：i18n 国际化

=== 目录结构

----
i18n/
└── en/
    └── docusaurus-plugin-content-docs/
        ├── current/          # 英文 current 版本文档
        │   ├── intro.adoc
        │   ├── getting-started.adoc
        │   ├── features.adoc
        │   └── asciidoc-showcase.adoc
        └── version-1.0.0/    # 英文 1.0.0 版本文档
            ├── intro.adoc
            └── ...
----

=== 工作原理

* 默认 locale（`zh-CN`）直接使用 `docs/` 目录的文件
* 其他 locale 的文档放在 `i18n/<locale>/docusaurus-plugin-content-docs/<version>/` 下
* 如果某个 locale 没有对应翻译文件，会 fallback 到默认 locale 的内容

=== 导航栏语言切换

在 `themeConfig.navbar.items` 中添加：

[source,javascript]
----
{ type: 'localeDropdown', position: 'right' }
----

== 第八步：多版本文档

=== 创建版本快照

[source,bash]
----
npx docusaurus docs:version 1.0.0
----

该命令自动执行：

* 复制 `docs/` -> `versioned_docs/version-1.0.0/`
* 复制 sidebars -> `versioned_sidebars/version-1.0.0-sidebars.json`
* 创建/更新 `versions.json`

之后 `docs/` 目录为 "Next"（开发中）版本。

|===

== 第九步：宽屏显示优化

默认 Docusaurus 的内容区域被 `max-width: 1140px` 限制，在 2560px 等宽屏显示器上两侧会有大量空白。

=== CSS 优化

[source,css]
----
/* 文档页面内容区撑满 */
[class*='docMainContainer'] .container {
  max-width: 100%;
}

/* 右侧 TOC 宽度限制 */
[class*='tableOfContents'] {
  max-width: 240px;
  font-size: 0.85rem;
}
----

=== 左侧边栏可隐藏

配置 `themeConfig.docs.sidebar.hideable: true`，左侧边栏底部出现折叠按钮（`<<`），点击可完全收起。

=== 右侧大纲（TOC）

右侧 TOC 不支持内置隐藏，但可通过以下方式控制：

* `themeConfig.tableOfContents.minHeadingLevel` / `maxHeadingLevel` — 控制显示的标题层级
* 单页面 frontmatter `hide_table_of_contents: true` — 隐藏该页面的 TOC
* 自定义 CSS 限制宽度（如上）

== 常见问题

=== `require.resolveWeak is not a function`

原因：`package.json` 中设置了 `"type": "module"`。删除该字段，配置文件改用 `.mjs` 扩展名。

=== webpack ProgressPlugin 报错

原因：webpack 5.106+ 对 ProgressPlugin 参数做了严格校验。解决：在 `overrides` 中锁定 `"webpack": "5.97.1"`。

=== AsciiDoc 页面内容为空

原因：`@asciidoc-js/docusaurus-asciidoc` 的 remark 插件在第一次编译后删除了注册表条目。解决：使用 patch-package 修复（见第四步）。

=== 搜索提示 "search index is only available when you run build"

正常现象。`@easyops-cn/docusaurus-search-local` 只在 `npm run build` 时生成索引。使用 `npm run build && npx docusaurus serve` 验证搜索功能。

=== 代码块无语法高亮（如 Java）

原因：Docusaurus 使用的 `prism-react-renderer` 默认只包含部分语言（bash, go, python, rust, javascript, typescript 等），`java` 不在默认列表中。

解决：在 `docusaurus.config.mjs` 的 `themeConfig.prism.additionalLanguages` 中添加需要的语言：

[source,javascript]
----
prism: {
  additionalLanguages: ['java', 'shell-session'],
}
----

=== 代码块不显示行号

原因：AsciiDoc 的 `[source,java,linenums]` 中的 `linenums` 属性需要转为 Docusaurus 识别的 `meta: "showLineNumbers"` 才能触发行号显示。需要 patch `@asciidoc-js/adast-util-to-mdast`（见第四步）。

=== 命令行会话代码块无高亮

AsciiDoc 中 `[source,console]` 的 `console` 不是 Prism 支持的语言名。对于 `$ command` 格式的终端会话，应使用 `[source,shell-session]` 并在 `additionalLanguages` 中添加 `'shell-session'`。

=== 表格单元格合并（colspan）不生效

原因：标准 mdast 的 `table` 节点不支持 `colspan`/`rowspan`。需要 patch `@asciidoc-js/adast-util-to-mdast`，使有合并单元格的表格走 `buildRichTable` 路径并输出 HTML `colspan`/`rowspan` 属性（见第四步）。

=== PlantUML 图片不显示

检查网络是否能访问 `kroki.io`。该服务为公网免费 API，内网环境可能需要配置代理或自建 Kroki 服务。
